# Testing Guidelines

This document details the strategies and checks for testing code changes.

## 1. Static Checks
Verify syntax, imports, types, and formatting using the built-in scripts:
```bash
# Run lint check
npm run lint

# Run TypeScript compilation check
npm run typecheck
```

## 2. Manual Verification
Test changes in actual runtime environments:
- **Web Browser**: Run `npm run dev` and press `w` to test in web environment.
- **Native Device**: Scan the QR code generated by `npm run dev` with the Expo Go app on a physical iOS/Android device.

## 3. Database & Network Mocking
- Test offline states and database failures by temporarily disabling the internet connection or modifying API environment variables.
- When introducing new database interactions, verify that appropriate tables exist in the Supabase instance.
